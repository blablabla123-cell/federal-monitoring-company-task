Необходимо разработать простое RESTful API для системы управления задачами (Task Manager). 

Приложение должно позволять пользователям регистрироваться, авторизовываться, создавать, просматривать, обновлять и удалять задачи. Каждая задача принадлежит конкретному пользователю (авторизация обязательна для операций с задачами). 

Для повышения производительности и обработки сложных задач добавьте кеширование и асинхронную обработку. 


 Технические требования:
• Используйте фреймворк Nest.js с TypeScript. 
• База данных: PostgreSQL. Допустимо использование ORM (например, TypeORM или Prisma). 
• API: RESTful эндпоинты. Реализуйте аутентификацию с использованием JWT. Обработайте авторизацию (пользователь может работать только со своими задачами). Добавьте обработку ошибок (custom errors, HTTP статусы).
• Интегрируйте Redis для кеширования часто запрашиваемых данных, таких как списки задач пользователя. Кэшируйте результаты GET-запросов (например, список задач) с TTL (time-to-live) не более 5 минут. Инвалидируйте кэш при обновлении или удалении задач. 
• Добавьте пример CPU-intensive операции, такой как генерация отчета о задачах (например, расчет статистики по задачам пользователя с использованием сложных вычислений). Для этого используйте очередь задач (например, Bull или @nestjs/bull с Redis в качестве брокера). Операция должна обрабатываться асинхронно: при запросе возвращайте немедленный ответ с ID задачи, а результат отправляйте пользователю по завершении (например, через WebSocket или email-уведомление; реализуйте хотя бы один вариант). 
• Напишите unit- и integration-тесты с использованием Jest (встроенного в Nest.js). Обеспечьте test-coverage не менее 70% (проверьте с помощью Jest coverage report). Тесты должны покрывать аутентификацию, CRUD-операции с задачами, кеширование и обработку очередей. 

 Дополнительно: 
 • Используйте Git для версионного контроля. Создайте репозиторий с несколькими коммитами, показывающими процесс разработки. 
 • Создайте Dockerfile для приложения и docker-compose.yml для запуска приложения с PostgreSQL. 
 • CI/CD: в README опишите, как настроить простой пайплайн. Пайплайн должен включать: запуск тестов, проверку coverage (fail если ниже 70%), build Docker-образа и deploy (в локальный registry или на тестовый сервер).
